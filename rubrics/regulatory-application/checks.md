# Regulatory Application — Automated Checks

These are deterministic checks. Each is binary pass/fail. Run these BEFORE applying the judgment rubric.

## Source Verification

- [ ] **Primary source cited.** Requirements reference the actual statute, administrative code, or issuing agency website — not just a blog post or secondary summary.
- [ ] **Issuing authority identified.** The specific government body that grants the license is named (e.g., "Illinois Department of Financial and Professional Regulation," not just "the state").
- [ ] **Statute/code numbers provided.** Specific legal references (e.g., "410 ILCS 705/") are included, not just general descriptions.
- [ ] **URLs or document references included.** Links to official application forms, fee schedules, or regulatory guidance are provided where available.

## Completeness

- [ ] **All application components listed.** Every document, form, fee, and requirement is enumerated — not summarized as "various documents."
- [ ] **Fees specified with amounts.** Dollar amounts for application fees, licensing fees, and any other costs are stated, not described vaguely.
- [ ] **Timeline included.** Application window dates, processing times, and/or expected approval timelines are stated.
- [ ] **Eligibility criteria listed.** Who can apply, disqualifying factors, and any special eligibility (e.g., social equity provisions) are addressed.
- [ ] **Entity requirements addressed.** What business entity type is required, registration requirements, and any in-state presence requirements.

## Structural Integrity

- [ ] **Checklist format.** Output is structured as an actionable checklist, not a narrative essay. Each item is a discrete task that can be checked off.
- [ ] **Dependencies identified.** Items that must be completed before other items are sequenced or explicitly noted (e.g., "entity formation must precede application submission").
- [ ] **Owner/responsible party suggested.** For each major task, it's clear who should do it (applicant, lawyer, accountant, etc.).
- [ ] **No contradictions.** Requirements don't contradict each other (e.g., listing two different fee amounts for the same thing).

## Actionability

- [ ] **Contact information included.** Phone number, email, or office address for the issuing authority is provided.
- [ ] **Next immediate action is clear.** The very first thing the applicant should do is explicitly identified.
- [ ] **Deadline awareness.** If there's an application window or deadline, it's prominently flagged with the specific dates and days remaining.
